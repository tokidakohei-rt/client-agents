# クライアントヘルススコア基準

## 概要

クライアントの健康状態を100点満点で評価するための基準定義です。

---

## スコア構成

| カテゴリ | 配点 | 説明 |
|---------|------|------|
| エンゲージメント | 25点 | コンタクト頻度、レスポンス |
| 商談進捗 | 25点 | ステージ進捗、滞留状況 |
| リスク要因 | 25点 | ネガティブシグナル（減点方式） |
| 拡大機会 | 25点 | アップセル/クロスセル可能性 |

---

## 評価基準詳細

### 1. エンゲージメント（25点）

#### コンタクト頻度

| スコア | 条件 |
|--------|------|
| 15-20点 | 週1回以上のコンタクト |
| 10-14点 | 2週に1回のコンタクト |
| 5-9点 | 月1-2回のコンタクト |
| 0-4点 | 月1回未満 |

#### レスポンス速度

| スコア | 条件 |
|--------|------|
| 5点 | 即日〜翌日レスポンス |
| 3点 | 3日以内レスポンス |
| 1点 | 1週間以内レスポンス |
| 0点 | 1週間以上レスポンスなし |

### 2. 商談進捗（25点）

#### ステージ進捗

| スコア | 条件 |
|--------|------|
| 15-20点 | 予定通りまたは前倒し |
| 10-14点 | 1週間以内の遅延 |
| 5-9点 | 2週間以内の遅延 |
| 0-4点 | 2週間以上の遅延 |

#### 滞留状況

| スコア | 条件 |
|--------|------|
| 5点 | 目標日数以内 |
| 3点 | 警告日数以内 |
| 0点 | 危険日数超過 |

### 3. リスク要因（25点）※減点方式

基本25点から減点

| 減点 | リスク要因 |
|------|-----------|
| -5点 | 軽微な懸念（予算検討中等） |
| -10点 | 中程度の懸念（競合検討中） |
| -15点 | 重大な懸念（ネガティブフィードバック） |
| -20点 | 危機的状況（解約検討明言） |

#### リスクシグナル例

```yaml
軽微:
  - 「予算を検討中」
  - 「上長に確認が必要」
  - レスポンス遅延（7日未満）

中程度:
  - 「他社も検討している」
  - 利用頻度の低下（20%以上）
  - MTGキャンセル

重大:
  - 「期待と違った」
  - クレーム発生
  - 30日以上コンタクトなし

危機的:
  - 「解約を検討している」
  - 正式なクレーム
  - 決裁者の交代（ネガティブ）
```

### 4. 拡大機会（25点）

| スコア | 条件 |
|--------|------|
| 20-25点 | 明確な追加ニーズ、予算確保済み |
| 15-19点 | 追加ニーズあり、予算検討中 |
| 10-14点 | 潜在的なニーズ |
| 5-9点 | ニーズ不明、現状維持傾向 |
| 0-4点 | 縮小傾向、ダウングレード検討 |

---

## ヘルスレベル分類

| スコア | レベル | 色 | 説明 | アクション優先度 |
|--------|--------|-----|------|-----------------|
| 80-100 | High | 🟢 | 健全、拡大機会あり | 低（維持・拡大） |
| 50-79 | Medium | 🟡 | 要注意、フォロー必要 | 中（定期フォロー） |
| 0-49 | Low | 🔴 | 危険、即時介入必要 | 高（即時対応） |

---

## スコア計算例

### 例1: 健全なクライアント（85点）

```
【エンゲージメント】
- コンタクト頻度: 週1回 → 18点
- レスポンス: 翌日 → 5点
→ 小計: 23/25点

【商談進捗】
- ステージ: 予定通り → 18点
- 滞留: 目標内 → 5点
→ 小計: 23/25点

【リスク要因】
- リスクなし → 25点
→ 小計: 25/25点

【拡大機会】
- 追加ニーズあり、予算検討中 → 14点
→ 小計: 14/25点

【総合】23 + 23 + 25 + 14 = 85点（High）
```

### 例2: 要注意クライアント（62点）

```
【エンゲージメント】
- コンタクト頻度: 2週に1回 → 12点
- レスポンス: 3日 → 3点
→ 小計: 15/25点

【商談進捗】
- ステージ: 1週間遅延 → 12点
- 滞留: 警告日数内 → 3点
→ 小計: 15/25点

【リスク要因】
- 競合検討中（-10点）→ 15点
→ 小計: 15/25点

【拡大機会】
- 潜在ニーズあり → 12点
→ 小計: 12/25点

【総合】15 + 15 + 15 + 17 = 62点（Medium）
```

### 例3: 危険なクライアント（35点）

```
【エンゲージメント】
- コンタクト頻度: 月1回未満 → 3点
- レスポンス: 1週間以上なし → 0点
→ 小計: 3/25点

【商談進捗】
- ステージ: 3週間遅延 → 5点
- 滞留: 危険日数超過 → 0点
→ 小計: 5/25点

【リスク要因】
- ネガティブフィードバック（-15点）→ 10点
→ 小計: 10/25点

【拡大機会】
- 縮小傾向 → 2点
→ 小計: 2/25点

【総合】3 + 5 + 10 + 17 = 35点（Low）
```

## データソース

### Pipedriveから取得

```yaml
エンゲージメント:
  - activities.last_activity_date
  - activities.count
  - notes.response_time（カスタムフィールド）

商談進捗:
  - deals.stage_id
  - deals.update_time
  - deals.expected_close_date

リスク要因:
  - notes.content（キーワード検出）
  - activities.outcome

拡大機会:
  - deals.value（変化）
  - notes.content（キーワード検出）
```

|------|------|

| 手動レビュー | 週次（Lowスコアは即時） |
| 基準見直し | 四半期 |
