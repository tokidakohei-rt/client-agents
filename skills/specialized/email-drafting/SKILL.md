# email-drafting

## Overview

クライアントへのビジネスメールを状況に応じて作成するスキル。

---

## Email Types

### 1. フォローアップメール

```yaml
trigger: 商談後、提案後、MTG後
tone: professional, warm
structure:
  - お礼
  - 要点の振り返り
  - 次のアクション
  - クロージング
```

### 2. アポイント依頼メール

```yaml
trigger: 新規アプローチ、再アプローチ
tone: professional, concise
structure:
  - 自己紹介（必要に応じて）
  - 目的
  - 候補日時
  - クロージング
```

### 3. 提案メール

```yaml
trigger: 提案送付
tone: professional, compelling
structure:
  - 課題の確認
  - 提案概要
  - ベネフィット
  - 次のステップ
```

### 4. リマインダーメール

```yaml
trigger: 返信なし、期日前
tone: professional, gentle
structure:
  - リマインド
  - 前回の要点
  - 再度のお願い
```

### 5. お礼メール

```yaml
trigger: 成約、契約更新
tone: warm, appreciative
structure:
  - お礼
  - これまでの振り返り
  - 今後の期待
```

## Output Format

```markdown
## メール作成: {PURPOSE}

### 宛先
- To: {EMAIL}
- Cc: {CC_EMAIL}（必要に応じて）

### 件名
{SUBJECT_LINE}

### 本文

---

{EMAIL_BODY}

---

### パーソナライズポイント
- {POINT_1}: {CUSTOMIZATION}
- {POINT_2}: {CUSTOMIZATION}

### 送信前チェック
- [ ] 宛先確認
- [ ] 件名は明確か
- [ ] 誤字脱字チェック
- [ ] 添付ファイル確認（該当する場合）
```

## Templates

### フォローアップメール

```
件名: 【お礼】{MTG内容}について

{NAME}様

お世話になっております。{COMPANY}の{SENDER}です。

先日は{MTG内容}にお時間をいただき、誠にありがとうございました。

{要点の振り返り}

{次のアクション}

ご不明な点がございましたら、お気軽にお問い合わせください。
引き続きよろしくお願いいたします。

{SIGNATURE}
```

### アポイント依頼メール

```
件名: {目的}のご相談

{NAME}様

お世話になっております。{COMPANY}の{SENDER}です。

{目的・背景}

つきましては、30分ほどお時間をいただき、
詳しくご説明させていただければと存じます。

以下の日程でご都合いかがでしょうか。

・{候補1}
・{候補2}
・{候補3}

上記以外でも調整可能ですので、
ご希望の日時をお知らせいただければ幸いです。

何卒よろしくお願いいたします。

{SIGNATURE}
```

### リマインダーメール

```
件名: 【再送】{元の件名}

{NAME}様

お世話になっております。{COMPANY}の{SENDER}です。

先日お送りしました{内容}について、
お忙しいところ恐れ入りますが、
ご確認いただけましたでしょうか。

{前回の要点の簡潔な再掲}

ご不明な点がございましたら、
お気軽にお問い合わせください。

お手数をおかけしますが、
ご検討のほどよろしくお願いいたします。

{SIGNATURE}
```

## Personalization Guidelines

### 1. 相手の情報を活用

```
- 名前を正確に
- 役職を適切に
- 過去のやり取りを参照
```

### 2. 状況に応じたトーン

```
- 新規: ややフォーマル
- 既存（関係良好）: やや親しみ
- クレーム後: 丁寧・謝意
```

### 3. 具体性を持たせる

```
- 具体的な日付・数字
- 前回の会話内容
- 相手の課題への言及
```

---

## Quality Checklist

### 送信前チェック

- [ ] 宛先は正しいか
- [ ] 件名は明確で適切か
- [ ] 本文の誤字脱字はないか
- [ ] トーンは適切か
- [ ] 次のアクションは明確か
- [ ] 添付ファイルは正しいか（該当時）
- [ ] CCは必要か、正しいか

---

## Best Practices

### DO ✅

- ✅ 簡潔で明確な文章
- ✅ 相手の名前を正確に
- ✅ 具体的なアクションを提示
- ✅ 適切なトーンを選択

### DON'T ❌

- ❌ 長すぎるメール
- ❌ 曖昧な表現
- ❌ 一方的な内容
- ❌ 誤字脱字
