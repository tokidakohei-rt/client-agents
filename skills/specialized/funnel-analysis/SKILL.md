# funnel-analysis

## Overview

ファネル全体の状況を分析し、歩留まり（コンバージョン率）を計算、ボトルネックを特定するスキル。

---

## Capabilities

### 1. ファネル概要分析

```
入力: Pipedrive Deals, Pipelines, Stages
出力: ステージ別の件数・金額・CVR
```

### 2. ボトルネック特定

```
入力: ステージ間CVR
出力: 最も改善が必要なステージ、推定原因
```

### 3. トレンド分析

```
入力: 時系列データ
出力: 前週比/前月比の変化、傾向
```

### 4. アラート生成

```
入力: 現在値、閾値
出力: 異常値アラート
```

## Metrics Calculation

### CVR（コンバージョン率）

```
CVR = (次ステージ件数 / 現ステージ件数) × 100
```

### 滞留日数

```
滞留日数 = 現在日 - ステージ移動日
```

### ファネル効率

```
効率 = (成約件数 / リード件数) × 100
```

## Thresholds（閾値）

### CVR目安

| ステージ遷移 | 目標CVR | 警告閾値 |
|-------------|---------|---------|
| Lead → Appointment | 30% | < 20% |
| Appointment → Opportunity | 50% | < 35% |
| Opportunity → Proposal | 60% | < 45% |
| Proposal → Won | 40% | < 25% |

### 滞留日数目安

| ステージ | 目標日数 | 警告閾値 |
|---------|---------|---------|
| Lead | 7日 | > 14日 |
| Appointment | 14日 | > 21日 |
| Opportunity | 30日 | > 45日 |
| Proposal | 14日 | > 21日 |

## Best Practices

### DO ✅

- ✅ 定量データに基づく分析
- ✅ 比較可能な指標を使用
- ✅ アクション可能な提案を含める
- ✅ トレンドを考慮

### DON'T ❌

- ❌ 主観的な判断のみ
- ❌ 文脈なしの数字の羅列
- ❌ アクションなしの報告
